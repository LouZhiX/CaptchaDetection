# 配置文件规范（L2）

## 1. 配置目录

`etc/` 目录下：

| 类型      | 用途                                         |
| --------- | -------------------------------------------- |
| `*.toml`  | 主要业务配置（推荐新增配置使用 toml）        |
| `*.ini`   | 历史遗留配置（尽量不新增）                   |
| `*.tmpl`  | 模板配置（pongo2 渲染）                      |
| `*.json`  | 少量结构化数据                               |

## 2. 命名

- 文件名：小写 + 短横线，`log-config.toml`、`server.toml`
- 键名：snake_case，`max_connections`、`timeout_ms`

## 3. 加载

- 统一通过 `pkg/configuration/` 加载
- **禁止** 在业务代码里直接 `os.Open("etc/xxx.toml")`

## 4. 新增配置项的步骤

1. 在对应 `etc/xxx.toml` 中添加键，**给默认值 + 中文注释**
2. 在 `pkg/configuration/` 对应结构体中添加字段
3. 在使用处通过 configuration 模块读取
4. 在 MR 描述中说明：新增配置、默认值、线上如何下发

## 5. 敏感值

- **禁止**在 `etc/*.toml` 中硬编码密码、AK、SK、token
- 改用环境变量或配置中心（KMS / 配置中心）注入

## 6. 默认值

- 每个配置项**必须有合理默认值**，避免"忘配置就起不来"
- 在 `pkg/configuration/` 的初始化阶段填充默认值

## 7. 配置变更的兼容性

- 重命名键名 → **保留旧键名至少一个版本**（双读）
- 删除键名 → 在 Release Note 中显著提示
- 修改默认值 → 评估对存量部署的影响

---

## AI 操作约束

- 修改 `etc/*.toml` 时，**必须保留同行或上方的中文注释**
- 修改 `etc/*.toml` 时，**必须同步修改** `pkg/configuration/` 对应字段
- 修改前先跑一次：`grep -n "<key_name>" pkg/ etc/` 确认所有使用点
